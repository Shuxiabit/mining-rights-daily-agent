# 设计决策记录

## ADR-001：使用三个独立 MCP Server

**决定**：新闻、PDF 和价格分别部署为独立 MCP Server。

**原因**：数据源、依赖和故障模式不同；独立服务更容易替换、测试和接入其他 MCP Client。

## ADR-002：模型 API 可选

**决定**：模型只负责基于结构化上下文组织文字，不负责制造事实。

**原因**：面试官无需准备密钥也能五分钟跑通；模型不可用时仍可验证完整数据流。

## ADR-003：显式数据状态

**决定**：所有响应必须声明 `live`、`demo` 或 `unavailable`。

**原因**：第三方免费接口不稳定。透明降级比“看起来成功”的错误数据更符合生产系统要求。

## ADR-004：价格代理必须披露

**决定**：缺少免费稳定的锂现货接口时，可使用公开 ETF 等代理，但必须在结果中说明。

**原因**：代理品种只能表达相关市场方向，不能冒充现货价格。

## ADR-005：保守 PDF 抽取

**决定**：只接受明确匹配分类、矿石量、品位和单位的文本行。

**原因**：矿业报告脚注和单位复杂。无法确认时返回 `unavailable`，优先避免错误确定性。

## 已知限制

- 当前 PDF 解析适合文本型、结构相对清晰的报告。
- 新闻正文受网站 HTML 结构和访问策略影响。
- 免费行情仅提供披露过的代理品种，不等同于矿产品现货报价。

## ADR-006：外部资源使用安全下载器

**决定**：网页与 PDF 仅允许公网 HTTP(S)，逐次校验重定向目标，并限制类型和大小。

**原因**：MCP 工具可能接收其他客户端提供的 URL，需要避免 SSRF、内网访问和超大响应。

## ADR-007：线上失败不自动伪装成演示成功

**决定**：只有显式离线模式或 `demo://` 输入可返回 DEMO；线上失败返回 `UNAVAILABLE`。

**原因**：演示稳定性不能高于事实完整性，调用者应明确看到数据源不可用。
